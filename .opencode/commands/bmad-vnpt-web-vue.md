# /bmad-vnpt-web-vue

Use the `bmad-vnpt-web-vue` skill and workflow.

Apply this when the task or repository is clearly Vue-based, especially when you see:
- `vue`, `vite`, `@vitejs/plugin-vue`
- `pinia`
- `vue-router`
- `.vue` single-file components
- `vite.config.ts` or `vite.config.js`

The skill should guide the work toward:
- Vue 3 + Composition API
- `<script setup>` SFCs
- Pinia only for shared/global state
- composable-first reuse
- router-aware route/view design
- Vitest + Vue Test Utils verification

# Python stack decision matrix

## Choose by task shape
- **FastAPI**: typed API/service, OpenAPI, dependency-driven service layers
- **Django**: server-rendered/full-stack/admin-heavy product
- **PySide6**: rich desktop app
- **tkinter**: tiny built-in GUI utility
- **Typer**: user-facing CLI with good DX
- **argparse**: tiny zero-dependency script
- **scikit-learn**: tabular/classical ML
- **PyTorch**: deep learning / tensor-heavy training

## Do not migrate blindly
Prefer the repository's current framework unless the task explicitly requests a migration.
